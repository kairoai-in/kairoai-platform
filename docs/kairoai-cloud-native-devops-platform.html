What Is KairoAI?

A GitHub-native AI review platform for Terraform pull requests.

How it works

  • Users sign in with GitHub OAuth.
  • They install the KairoAI GitHub App on selected repos or orgs.
  • PR webhooks trigger Terraform validation, security, cost, governance, and AI analysis.
  • Results appear in GitHub checks, PR comments, annotations, and the dashboard.

What it produces

  • Blocking and non-blocking findings.
  • Provider-aware security checks with Checkov.
  • Cost deltas and resource-level cost breakdowns.
  • Governance verdicts for naming, tags, regions, and module usage.
  • AI-generated recommendations and merge decision support.

Security Architecture

Identity, secrets, network boundaries, and policy gates are layered across app and infra.

Identity and access

  • GitHub OAuth authenticates dashboard users.
  • GitHub App installations scope repository access per user/org.
  • GitHub Actions uses Azure OIDC instead of long-lived Azure secrets.
  • AKS uses managed identities and RBAC for cloud integration.

Secrets and network

  • Runtime secrets live in Azure Key Vault and sync to AKS through the secrets runtime path.
  • PostgreSQL is Azure Flexible Server, not a pod.
  • Key Vault private endpoints are live across hub, test, prod, and prod-DR.
  • App Gateway WAF is currently in Detection mode for prod to avoid false-positive 403s while retaining logs.

Current report-only controls include Checkov soft-fail in Terraform PRs, Trivy SARIF evidence, Snyk reports, and OWASP ZAP DAST artifacts.

Remote State And Secret Management

Terraform state is isolated by environment while Azure auth stays secretless.

Remote state

  • Hub subscription stores Terraform state centrally.
  • Containers: `hubtfstate`, `testtfstate`, and `prodtfstate`.
  • Keys isolate hub, bootstrap, test, prod, and prod-DR roots.
  • Azure Blob backend locking protects against concurrent mutation.

Pipeline identity

  • GitHub Actions exchanges OIDC tokens with Entra ID.
  • No Azure client secret is stored in GitHub.
  • Terraform identity has subscription Contributor and tfstate blob permissions.
  • User Access Administrator is only for role-assignment operations where required.

Target branches map to environments: `hub` -> hub subscription, `test` -> test subscription, and `main` -> prod plus prod-DR.

Observability And DAST

Runtime health, platform telemetry, and external security testing are part of the platform story.

Monitoring stack

  • Azure Managed Prometheus for AKS metrics.
  • Azure Managed Grafana for dashboards.
  • Log Analytics and Application Insights for logs and app telemetry.
  • Azure Monitor alerts for AKS, PostgreSQL, App Gateway, and Front Door signals.

DAST evidence

  • OWASP ZAP runs from GitHub Actions against dashboard, API, and Argo CD routes.
  • Reports are uploaded as HTML, Markdown, and JSON artifacts.
  • Test and prod targets can be scanned separately through workflow inputs.
  • Results support demo evidence and future hard gates.
